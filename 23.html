The size solution is apparent once one realizes that workers attempting to move into each other's squares will swap positions. The speed solution, however, is a "hack" that exploits the fact that failed cases are not counted when computing the average speed. The trick is to make the cases that take longer fail by unrolling the loop and only completing a fixed number of iterations. The number of iterations may need to be adjusted along with getting lucky test data in order to get the target value of 16. It may be possible to get 15 or lower with particularly lucky data.
