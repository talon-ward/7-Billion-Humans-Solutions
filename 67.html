Size

   01| step S
   02| pickUp H
   **| jmp01:
   03| step S
   04| if H == "button"
   05|| step N
   06|| tell W "Ok!"
   07|| listen "Go!"
   08|| jump jmp01
   **| jmp02:
   09| drop
   10| if W != "a worker"
   11|| tell "everyone" "Go!"
   12| listen "Ok!"
   13| tell W "Ok!"
   14| "mem 2" = calc E / "5"
   15| "mem 1" = calc H x "2"
   16| if "mem 1" > "9"
   17|| "mem 1" = calc "mem 1" - "10"
   18| "mem 1" = calc "mem 1" + "mem 2"
   19| pickUp H
   20| write "mem 1"
   21| jump jmp02

The key insight for the size solution is that the carry digit can be computed in a single "calc" operation by taking the value of the adjacent digit and dividing by 5. The speed solution converts the "calc" operations into a binary tree. It might have some room to be made faster.
