The size solution is fairly straightforward, just implementing the standard addition algorithm using standard optimizations and synchronizing with "listen" and "tell" commands. The speed solution is "hacky" in that it uses fine-tuned timing. Since the ones digit worker is busy the entire time with no delays, this solution should be incredibly close to the theoretical optimal.
