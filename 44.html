All the solutions to this puzzle are "hacks." The size solution is so small you basically have no choice, but it is upgraded to a difficulty 2, since it's not guaranteed to succeed. Remove the E and SE options for a higher success rate, as it can fail if all of a certain number wander off into a hole. Although, this still does not guarantee success because there are some pathing issues that can trigger all of a certain number to rush into a hole. The speed solution will work with 100% accuracy if the first two self-destruct "if" statements are removed, but the average speed becomes about 27 secs. This is the fastest solution I know with 100% accuracy. There is an incredibly "hacky" solution that has an average speed of 1 secs. I did not come up with this solution, however, so I did not include it, but be aware it exists.
