The trick with the size challenge is that the objective gets accomplished before the bottom left worker has time to write "100" to the data cube. The speed solution is incredibly obnoxious to enter. It involves worker isolation and replacing a "calc" command with a (massive) binary tree. A naive approach will fail due to maximum program length. To compensate, the addition for the last three columns is done using the "calc" command, and the first six additions are performed by the binary tree.
