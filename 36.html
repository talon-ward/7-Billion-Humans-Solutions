The size solution is very simple: just add a jump to the previous ("Seek and Destroy 1") solution. The speed solution, however, is quite difficult. It requires various optimizations, such as isolating individual workers. However, maximum program length is an issue, so the focus must remain on the two workers who have to travel the farthest, while similarities in other workers can lead to code reuse. Memory location use is maximized with "mem 2" containing the smallest and "mem 3" the next smallest datacubes in each pass, cutting the number of iterations in half (rounding up).
